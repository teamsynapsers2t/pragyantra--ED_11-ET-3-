### Task 6 — Row Level Security

#### [NEW] [rls_policies.sql](file:///c:/Users/tumra/Documents/paper-AI/paper-ai/sql/rls_policies.sql)
SQL script that:
1. Enables RLS on: `attempts`, `concept_mastery`, `weakness_signals`, `weakness_reports`, `sessions`, `interaction_events`
2. Creates policies for student self-access (`user_id = auth.uid()` mapped from Clerk JWT)
3. Makes engine functions `SECURITY DEFINER`: `fn_apply_attempt`, `fn_detect_root_flaws`, `fn_generate_weakness_report`
4. Sets read-only policies on reference tables: `questions`, `question_options`, `concepts`, `chapters`, `concept_prerequisites`, `question_concepts`, `question_stats`
5. Verifies the trigger still fires under RLS

> [!WARNING]
> The Clerk-to-Supabase user mapping uses a deterministic MD5 hash (`clerkIdToUuid`). The RLS policies need to match this UUID against the JWT claims. This requires the Supabase JWT template to include the Clerk user ID, and the policy to call a mapping function or match directly against `user_id`. I'll use a Supabase `auth.uid()` approach where the service-role client bypasses RLS for the trigger functions.
